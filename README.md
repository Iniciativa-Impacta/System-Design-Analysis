# Lib Virtual - Plataforma de Assinatura de Livros Digitais

## Descricao do Projeto

A Lib Virtual e uma plataforma de assinatura de livros digitais que permite que leitores acessem um catalogo amplo de obras mediante assinatura, enquanto autores e editoras sao remunerados proporcionalmente ao consumo real de leitura de seus titulos.

Inspiracao: O conceito e inspirado no modelo de "Netflix de livros", mas adaptado ao contexto editorial brasileiro, com foco em remuneracao justa aos criadores de conteudo e acessibilidade aos leitores.

Este projeto foi desenvolvido como parte da disciplina de Analise e Modelagem de Sistemas, aplicando conceitos de engenharia de requisitos, SRS e modelagem UML em um caso real de desenvolvimento de software.


## Visao do Produto

A Lib Virtual e uma solucao que resolve tres problemas principais:

1. Para Leitores: Acesso acessivel a um vasto catalogo de livros digitais
2. Para Autores: Remuneracao continua baseada no consumo real de suas obras
3. Para Editoras: Novo canal de distribuicao e monetizacao digital

### Problema

- Custo elevado de livros digitais para estudantes e leitores frequentes
- Autores independentes com dificuldade para distribuir e monetizar suas obras
- Editoras buscando alternativas a venda unitaria de e-books
- Descoberta de conteudo limitada e dispersa em multiplas plataformas

### Publico-Alvo

- Leitores Frequentes: Estudantes, profissionais e entusiastas de leitura
- Autores Independentes: Escritores buscando distribuicao e monetizacao
- Editoras: Empresas buscando novos canais digitais

### Proposta de Valor

Para Leitores:
- Acesso ilimitado ao catalogo mediante assinatura
- Recomendacoes personalizadas
- Leitura sincronizada entre dispositivos

Para Autores e Editoras:
- Remuneracao proporcional a leitura
- Dashboard analitico com metricas de desempenho
- Protecao contra pirataria

## Modelo de Negocio

A plataforma opera com modelo de assinatura recorrente, onde leitores tem acesso ilimitado ao catalogo. A receita e distribuida proporcionalmente entre autores e editoras baseando-se no consumo efetivo de leitura.

## Conceitos Principais

### Modelagem de Software
- Reduz erros e ambiguidades no projeto
- Fornece base para design e implementacao
- Permite evoluir e adaptar o sistema

### Requisitos de Software
- Funcionais: O que o software deve fazer
- Nao-Funcionais: Como o software deve fazer
- Restricoes: Limitacoes e imposicoes

### SRS
Documento que especifica de forma clara e estruturada os requisitos do sistema, servindo como contrato entre stakeholders e desenvolvedores.
