# PARTE 1 - Introdução e Necessidade de Modelagem

## Apresentação da Disciplina

### Análise e Modelagem de Sistemas

A disciplina de **Análise e Modelagem de Sistemas** é fundamental para a formação de profissionais de tecnologia que desejam compreender como transformar requisitos de negócio em soluções de software robustas e escaláveis.

## Por que Modelagem?

### A Necessidade de Modelagem

A modelagem de sistemas é essencial porque:

1. **Instrumento para Viabilizar Software**
   - Possibilita a materialização de ideias em especificações concretas
   - Fornece um caminho entre o conceito e a implementação

2. **Conhecer e Exercitar Modelagem de Software**
   - Compreensão da Linguagem de Modelagem Unificada (UML)
   - Aplicação prática em projetos reais

3. **Aplicação em Projetos Reais**
   - Uso de metodologias consolidadas
   - Integração com ferramentas profissionais

## Modelos de Software

### O que é um Modelo?

Um modelo é uma **abstração da realidade** que captura aspectos de interesse para uma determinada situação. Em engenharia de software, os modelos representam:

- **Estrutura:** Como o sistema é organizado
- **Comportamento:** O que o sistema faz
- **Interações:** Como as partes se comunicam

### Lacuna entre Requisitos e Código

Existe uma enorme lacuna entre o "requisito" (descrição de alto nível) e o "código fonte" (implementação técnica):

```
Requisitos ─────────────────────────── Código
     ↑                                    ↑
   Alto Nível                      Muito Detalhado
   Ambíguo                          Complexo
   Não-técnico                      Específico
   
              ↓ NOSSAS MODELAGENS ↓
        Mais detalhadas que requisitos
        Menos complexas que o código
        Representações gráficas
        Facilitam decisões de design
```

## Benefícios da Modelagem

### 1. Clareza
- **Tradução de requisitos em representações visuais claras**
- Redução de ambiguidades
- Comunicação eficiente entre stakeholders

### 2. Economia de Custos e Tempo
- **Identificação de problemas antes da implementação**
- Evita retrabalho
- Reduz surpresas durante o desenvolvimento

### 3. Colaboração
- **Comunicação eficiente entre equipes multidisciplinares**
- Facilita a discussão de soluções
- Integra diferentes perspectivas

### 4. Escalabilidade
- **Maior facilidade para evoluir e adaptar o Produto de Software**
- Suporta mudanças futuras
- Documenta o sistema para manutenção

## Motivadores da Modelagem

Diversos fatores impulsionam a necessidade de modelagem:

### Fatores Técnicos
- Requisitos multifacetados
- Protótipos e validações
- Tamanho e complexidade do projeto

### Fatores Organizacionais
- Prazos definidos
- Limitações de orçamento
- Documentação e manutenção

### Fatores Dinâmicos
- **Clientes solicitam modificações e melhorias**
  - Novos requisitos surgem frequentemente
  - Necessidade de evolução contínua

- **Mercado adota novas estratégias**
  - Competição dinâmica
  - Mudanças tecnológicas

- **Governo altera leis, impostos e alíquotas**
  - Conformidade regulatória
  - Adaptação a novas normativas

## Modelar é também Documentar!

Um aspecto fundamental é que **a modelagem serve como documentação** do sistema:

- Representações visuais facilitam compreensão
- Diagrama UML comunica idéias rapidamente
- Documentação gráfica perdura
- Reduz dependência de documentação textual

## Modelos de Software para Tomada de Decisões

Os modelos não são apenas artefatos de documentação, mas **ferramentas para tomada de decisões**:

- Avaliar arquiteturas alternativas
- Identificar riscos técnicos
- Validar soluções antes da implementação
- Comunicar riscos e oportunidades
