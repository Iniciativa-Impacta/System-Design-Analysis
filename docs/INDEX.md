## Ordem de Leitura Recomendada

### 1. **PARTE 1 - Introdução e Necessidade** [PARTE1.md](PARTE1.md)
**Duração: ~15 minutos**

Compreenda **por quê** a modelagem é importante:
- O que é modelagem de software
- Benefícios da modelagem
- Lacuna entre requisitos e código
- Motivadores e drivers

**Saiba Mais:** Conceitos fundamentais de sistema de software

---

### 2. **PARTE 2 - Engenharia de Requisitos** [PARTE2.md](PARTE2.md)
**Duração: ~20 minutos**

Aprenda **como** trabalhar com requisitos:
- Definição de Engenharia de Requisitos
- Tipos de requisitos (Funcionais, Não-Funcionais, Restrições)
- Casos de Uso
- Processo de elicitação e análise

**Saiba Mais:** Técnicas e processos para trabalhar com requisitos

---

### 3. **PARTE 3 - SRS e Artefatos** [PARTE3.md](PARTE3.md)
**Duração: ~20 minutos**

Domine a **documentação formal** de requisitos:
- O que é SRS
- Propósito e características
- Estrutura do documento
- Rastreabilidade de requisitos
- Fluxo de artefatos em engenharia

**Saiba Mais:** Padrões e templates para documentação profissional

---

### 4. **PARTE 4 - Exercício Prático** [PARTE4.md](PARTE4.md)
**Duração: ~60 minutos (Dinâmica em sala)**

**Aplique** tudo na prática com o exercício:
- Sistema de Gerenciamento de Biblioteca
- Dinâmica de papéis (Stakeholders e Analistas)
- Processo passo-a-passo
- Exemplos de resultados

**Saiba Mais:** Simulação realista de projeto de software

---

### 5. **PARTE 5 - Checklist de Validação** [PARTE5.md](PARTE5.md)
**Duração: ~25 minutos**

**Valide** e melhore a qualidade:
- Critérios de validação de requisitos
- Consistência e rastreabilidade
- Concisão e padrão
- Exemplos de contra-exemplos

**Saiba Mais:** Como garantir qualidade profissional no SRS

---

## Competências Desenvolvidas

Ao completar este curso, você será capaz de:

**Compreender** a importância da modelagem de software
**Identificar** tipos de requisitos e suas características
**Elicitar** necessidades de stakeholders
**Documentar** especificações de software profissionais
**Validar** requisitos usando critérios de qualidade
**Colaborar** efetivamente em projetos de engenharia de software
**Aplicar** UML para modelagem de sistemas
**Gerenciar** rastreabilidade de requisitos
